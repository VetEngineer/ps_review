# 제품 요구사항 정의서 (PRD)
**프로젝트명:** 개인화 가중치 기반 AI 앱 추천 및 비교 서비스 (Smart App Picker)
**문서 버전:** 1.0
**작성일:** 2025년 12월 08일

---

## 1. 상세 제품 설명 (Detailed Product Description)

### 1.1 배경 및 목적
현재 구글 플레이스토어와 애플 앱스토어는 수백만 개의 앱을 제공하지만, 사용자는 자신에게 딱 맞는 앱을 찾기 위해 수동으로 검색, 상세 페이지 진입, 리뷰 확인, 뒤로 가기를 반복하는 **'앱 선택 피로(App Selection Fatigue)'**를 겪고 있습니다. 기존 스토어의 랭킹 시스템은 대중적인 인기도만 반영할 뿐, "광고가 없고", "특정 기능이 있으며", "UI가 직관적인" 앱을 원하는 개인의 구체적인 니즈를 해결해주지 못합니다.

본 서비스는 **LLM(Large Language Model) 기반의 개인화 가중치 분석**을 통해, 사용자의 자연어 요구사항을 분석하고 여러 후보 앱을 **단일 비교표(Comparison Table)**로 시각화하여 제공합니다. 이를 통해 탐색 시간을 획기적으로 단축하고 사용자의 의사결정 확신을 높이는 것을 목표로 합니다.

### 1.2 해결하고자 하는 문제 (AS-IS vs TO-BE)

| 구분 | AS-IS (기존 스토어 탐색) | TO-BE (본 서비스 도입 후) |
| :--- | :--- | :--- |
| **탐색 방식** | 키워드 검색 후 리스트 나열 (단순 랭킹순) | 자연어 질문 기반 맥락(Context) 검색 |
| **비교 방식** | 앱 A 클릭 → 리뷰 읽기 → 뒤로 가기 → 앱 B 클릭 (Memory 의존) | **AI 자동 비교표**를 통해 한 화면에서 스펙 및 장단점 병렬 비교 |
| **정보 처리** | 사용자가 수백 개의 리뷰를 직접 읽고 필터링 (정성적) | AI가 리뷰 감성 및 기능 명세를 분석하여 **정량적 점수**로 변환 |
| **결과** | 정보 과부하로 인한 피로감, 확신 없는 다운로드 | 개인화된 추천 점수 확인 후 **확신에 찬 다운로드** |

## 2. 유사 서비스 분석 및 차별점 (Reference Services)

기존 앱 마켓 및 비교 서비스의 한계를 분석하여 본 서비스의 차별화 포인트를 도출했습니다.

### 2.1 주요 레퍼런스

| 서비스명 | 주요 특징 | 한계점 | 본 서비스의 차별화 (Rationale) |
| :--- | :--- | :--- | :--- |
| **Google Play / App Store** | AI 리뷰 요약 기능 도입 (단일 앱 대상). | 개별 앱의 요약만 제공할 뿐, **여러 앱 간의 상대적 비교**가 불가능함. | 경쟁 앱 3~4개를 선정하여 **비교표(Table)** 형태로 제공, 우위 요소를 시각화함. |
| **Slant / Versus.com** | 사용자 투표 기반의 소프트웨어/하드웨어 비교. | 데이터가 정적이며, 실시간 리뷰 데이터(최신 버그, 업데이트) 반영이 느림. | 실시간 크롤링 및 분석을 통해 **최신 리뷰(최근 6개월)** 기반의 현행화된 데이터 제공. |
| **AppBrain / data.ai** | 앱 데이터 분석 및 순위 제공 (B2B 중심). | 일반 사용자가 이해하기 어려운 마케팅 데이터 중심. | **20대 일반 사용자(Gen Z)**가 중요시하는 UI/UX, 광고 유무, 감성 품질에 집중. |

### 2.2 시장 내 포지셔닝
* **Target:** 20대 (효율성을 중시하며, 광고/나쁜 UI에 민감한 디지털 네이티브).
* **Niche:** "검색"과 "설치" 사이의 "검증 및 비교" 단계를 자동화하는 **'쇼핑 어시스턴트'** 포지션.

## 3. 핵심 기능 및 상세 명세 (Core Features & Specifications)

### 3.1 [입력] 자연어 검색 및 페르소나 가중치 설정
* **기능 설명:** 사용자가 구어체로 원하는 앱의 조건을 입력하면 AI가 의도를 파악하고 가중치를 설정합니다.
* **상세 명세:**
    * **Input:** 텍스트 입력창 (예: "아이패드에서 필기감 좋고 PDF 내보내기 무료인 노트 앱 찾아줘")
    * **Logic:** 입력된 문장에서 핵심 키워드(필기감, PDF 무료) 추출 및 가중치(Persona Weight, $\omega$) 자동 할당.
    * **User Option:** 검색 후 사용자가 슬라이더 UI 등을 통해 가중치 직접 수정 가능 (예: '무료 여부' 중요도 상향 조정).

### 3.2 [분석] 리뷰 감성 분석 및 기능 점수화 (Backend)
* **기능 설명:** 후보 앱들의 리뷰와 설명 텍스트를 분석하여 정량적인 점수를 산출합니다.
* **상세 명세:**
    * **Data Source:** 앱 스토어 리뷰(최근 6개월), 앱 상세 설명, 업데이트 로그.
    * **Sentiment Analysis:** NLP 모델을 활용하여 리뷰의 긍정/부정 비율 분석 (버그, 광고, 발열, 기능 오류 등).
    * **Feature Extraction:** OCR(스크린샷 분석) 및 텍스트 마이닝을 통해 실제 기능 유무(다크모드, 백업 지원 등) 판별.
    * **Scoring Algorithm:**
        $$\text{Final Score} = \sum (\text{Feature Score} \times \text{Persona Weight}(\omega))$$

### 3.3 [출력] AI 기반 동적 비교표 (Dynamic Comparison Table)
* **기능 설명:** 최종 선정된 Top 3~5개 앱의 특징을 한눈에 비교할 수 있는 표를 렌더링합니다.
* **상세 명세:**
    * **Format:** 행(기능/특징) x 열(앱 이름) 형태의 매트릭스 UI.
    * **Content:**
        * 최종 추천 점수 (AI Score)
        * 핵심 장점 요약 (3줄 요약)
        * 주요 불만 사항 (Warning: 예 - "최근 업데이트 후 로그인 오류 있음")
        * 주요 기능 지원 여부 (O/X 또는 등급 표시)
    * **Interaction:** 표 내 항목 클릭 시 해당 리뷰 원문으로 이동하거나 상세 근거 표시.

## 4. 추가 제안 기능 (Suggested Additional Features)

본 프로젝트의 타겟인 20대의 특성과 웹 플랫폼의 확장성을 고려한 추가 기능 제안입니다.

1.  **유사 앱 가격 변동 추적 (Price History):**
    * 유료 앱의 경우, 할인 기록을 보여주어 "지금 사는 것이 이득인지" 판단 돕기.
2.  **'이 앱을 쓴 사람들이 갈아탄 앱' 추천:**
    * 리뷰 데이터에서 "A앱 쓰다가 B앱으로 넘어왔는데 좋아요"라는 패턴을 분석하여 마이그레이션 경로 추천.
3.  **Youtube 리뷰/Shorts 연동:**
    * 텍스트 리뷰 외에 해당 앱을 다룬 인플루언서의 영상 링크를 함께 제공 (20대 타겟 맞춤).
4.  **원클릭 설치 링크 통합:**
    * 웹에서 비교 후 스마트폰으로 바로 다운로드 링크를 보낼 수 있는 QR 코드 제공.

## 5. 사용자 페르소나 및 시나리오 (User Persona & Scenarios)

### 5.1 타겟 페르소나: "효율 추구형 대학생 김민수 (23세)"
* **특성:** 디지털 네이티브, 시간 낭비를 극도로 싫어함, '갓성비' 중시, 광고 포함 여부에 민감.
* **Needs:** 과제용 스캐너 앱이 필요한데, 유료 결제 유도가 너무 심한 앱은 거르고 싶음.

### 5.2 사용자 시나리오 (User Journey)

| 단계 | 사용자 행동 | 시스템 동작 |
| :--- | :--- | :--- |
| **1. 접속 및 입력** | "워터마크 없고 스캔 속도 빠른 무료 스캐너 앱 추천해줘" 입력 | 자연어 처리로 키워드(워터마크X, 속도, 무료) 추출 및 가중치 설정. |
| **2. 결과 대기** | 로딩 화면 대기 (약 3~5초) | DB 조회 → 리뷰 크롤링/분석 → 점수 계산 ($\sum Feature \times \omega$). |
| **3. 비교 확인** | Top 3 앱(Adobe Scan, vFlat, CamScanner) 비교표 확인 | 앱별 '워터마크 유무', 'OCR 정확도 점수', '최근 유저 불만' 표시. |
| **4. 상세 조정** | vFlat이 좋아 보이나 '페이지 제한'이 걱정되어 상세 정보 클릭 | 해당 기능에 대한 구체적인 정책 및 관련 리뷰 요약 팝업 제공. |
| **5. 결정 및 이동** | 확신을 갖고 vFlat 선택 후 스토어 이동 버튼 클릭 | 해당 앱 스토어 페이지로 리다이렉트 (QR 코드 또는 링크). |

## 6. 기술 스택 권장사항 (Technical Stack Recommendations)

웹 플랫폼 기반으로 빠르고 정확한 AI 분석을 제공하기 위한 기술 스택입니다.

### 6.1 Frontend (Web)
* **Framework:** **Next.js (React)** - SEO 최적화 및 빠른 렌더링 속도 확보.
* **UI Library:** **Tailwind CSS** - 모던하고 반응형 디자인(모바일/데스크탑)을 신속하게 구축.
* **Visualization:** **TanStack Table** (효율적인 비교표 구현) 또는 **Chart.js** (점수 시각화).

### 6.2 Backend & AI
* **Language:** **Python** - 풍부한 AI/NLP 라이브러리 활용.
* **Framework:** **FastAPI** - 비동기 처리 지원으로 크롤링 및 AI 추론 속도 최적화.
* **NLP/LLM:**
    * **OpenAI API (GPT-4o mini 등)** 또는 **LangChain**: 복잡한 리뷰 요약 및 자연어 의도 파악.
    * **Hugging Face Transformers**: 비용 절감을 위한 감성 분석(Sentiment Analysis) 경량화 모델 호스팅.
* **Crawling:** **Selenium** 또는 **Playwright** - 동적 웹 페이지(스토어 리뷰) 데이터 수집.

### 6.3 Infrastructure & Database
* **Database:** **PostgreSQL** - 앱 메타데이터 및 사용자 로그 저장 (관계형 데이터).
* **Vector DB:** **Pinecone** 또는 **Milvus** - 리뷰 텍스트의 임베딩 벡터 저장 (유사 의미 검색 및 빠른 분석 용도).
* **Cloud:** **AWS** (Lambda - 크롤러 구동, EC2 - API 서버) 또는 **GCP**.

## 7. 비즈니스 모델 및 KPI (BM & KPI)

### 7.1 비즈니스 모델 (BM)
* **제휴 마케팅 (Affiliate):** 추천된 유료 앱 다운로드 또는 구독 발생 시 수수료 수취.
* **개발사 대상 리포트 판매 (B2B):** "경쟁 앱 대비 우리 앱의 취약점(리뷰 분석)"을 분석한 심층 리포트 제공.

### 7.2 핵심 성과 지표 (KPI)
* **유효 클릭률 (CTR):** 비교표 조회 후 실제 스토어 다운로드 버튼 클릭 비율.
* **탐색 시간 단축:** 기존 스토어 평균 탐색 시간 대비 본 서비스 이용 시 소요 시간 (목표: 50% 단축).
* **사용자 재방문율 (Retention):** 새로운 앱을 찾을 때 다시 서비스를 찾는 비율.
