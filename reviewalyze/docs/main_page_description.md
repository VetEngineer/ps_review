# 메인 페이지 Description

**화면명:** 메인 페이지 (Home Page)  
**버전:** 1.0  
**작성일:** 2025년 12월 08일

---

## 1. 화면 개요

사용자가 앱 키워드를 검색하고, 검색 결과에 따라 분석 결과를 표시하는 통합 대시보드 화면입니다. 검색 전 상태와 검색 후 상태가 동일한 페이지에서 동적으로 전환되며, 앱 리뷰 분석 결과를 시각적으로 제공합니다.

**Core Purpose:** 사용자가 원하는 앱을 키워드로 검색하고, AI가 분석한 3개 앱의 비교 결과를 한 화면에서 확인할 수 있도록 지원합니다.

## 3. 주요 컴포넌트 상세

### 3.1 배경 레이어

**목적:** 시각적 깊이감과 현대적인 느낌 제공

**구성:**

1. **레이어 1: Radial Gradient**
   - 상단 좌측: `rgba(99,102,241,0.08)` (indigo 계열)
   - 상단 우측: `rgba(16,185,129,0.08)` (emerald 계열)
   - 투명도: 8% (은은한 효과)

2. **레이어 2: Grid Pattern**
   - 패턴: 점선 격자 (`linear-gradient`)
   - 크기: `120px x 120px`
   - 색상: `rgba(148,163,184,0.15)`
   - 투명도: 40%

3. **레이어 3: Gradient Overlay**
   - 상단에서 하단으로: `from-white/85 via-white to-white/95`
   - 배경을 덮어 가독성 확보

**스타일:**
- 위치: `absolute`, `inset-0`
- z-index: 배경 레이어는 가장 하단

### 3.3 검색 섹션 (AppKeywordSearch)

**위치:** 헤더 하단  
**목적:** 사용자가 앱 키워드를 입력하고 검색 실행

**구성 요소:**

1. **카드 헤더**
   - 아이콘 컨테이너: `bg-indigo-50`, `rounded-lg`, `p-2`
   - 아이콘: `Hash` (indigo-600), `h-5 w-5`
   - 제목: "앱 키워드 검색"
   - 설명: "검색할 앱의 키워드를 입력하세요"

2. **검색 입력 영역**
   - 입력창 (`Input`):
     - 플레이스홀더: "예: 노트, 스캐너, 할일관리..."
     - 크기: `text-base`
     - 상태: `disabled` (검색 중 또는 비활성화 시)
     - 키보드 이벤트: Enter 키로 검색 실행
   
   - 검색 버튼 (`Button`):
     - 스타일: `bg-gradient-to-r from-indigo-500 via-blue-500 to-cyan-400`
     - 텍스트 색상: `text-white`
     - 그림자: `shadow-lg`, 호버 시 `shadow-xl`
     - 크기: `size="lg"`
     - 상태:
       - 검색 중: 로딩 스피너 (`Loader2`, `animate-spin`) + "검색 중..."
       - 일반: 검색 아이콘 (`Search`) + "검색"
     - 비활성화 조건: `disabled || isSearching || !keyword.trim()`

3. **검색 예시 영역**
   - 라벨: "검색 예시:" (`text-xs`, `font-semibold`, `text-slate-600`)
   - 예시 키워드 버튼:
     - 키워드: ['노트', '스캐너', '할일관리', '일기', '메모']
     - 스타일: `rounded-md`, `border-slate-200`, `bg-white`
     - 패딩: `px-3 py-1.5`
     - 텍스트: `text-xs`, `text-slate-600`
     - 호버: `hover:bg-slate-50`, `hover:border-indigo-300`
     - 클릭 시: 해당 키워드가 입력창에 자동 입력
     - 비활성화: 검색 중 또는 비활성화 시

**카드 스타일:**
- `Card` 컴포넌트
- 본문 간격: `space-y-4`
- 입력 영역: `flex`, `gap-2`

**인터랙션:**

1. **검색 실행**
   - 버튼 클릭 또는 Enter 키 입력
   - 키워드 유효성 검사 (빈 값 체크)
   - 검색 중 상태로 전환
   - `onSearch` 콜백 호출

2. **에러 처리**
   - 빈 키워드: Toast 알림 ("키워드를 입력해주세요")
   - 검색 실패: Toast 알림 ("검색 실패")

3. **예시 키워드 클릭**
   - 해당 키워드가 입력창에 자동 입력
   - 검색은 실행되지 않음 (사용자가 버튼 클릭 필요)

## 4. 데이터 흐름

### 4.1 상태 관리

```typescript
const [isAnalyzing, setIsAnalyzing] = useState(false);
const [analysisResults, setAnalysisResults] = useState<AnalysisResult[]>([]);
const [appName, setAppName] = useState<string>('');
const [comparisonData, setComparisonData] = useState<ComparisonData[]>([]);
```

### 4.2 검색 프로세스

1. **사용자 입력**
   - 키워드 입력 또는 예시 키워드 클릭
   - Enter 키 또는 검색 버튼 클릭

2. **검색 실행**
   - `handleKeywordSearch` 함수 호출
   - `isAnalyzing = true` 설정
   - 기존 결과 초기화 (`setAnalysisResults([])`, `setComparisonData([])`)

3. **데이터 처리**
   - 백엔드 API 호출 (현재는 모의 데이터)
   - 분석 결과 데이터 생성
   - 비교 데이터 생성 (3개 앱)

4. **결과 표시**
   - `setAnalysisResults(mockResults)`
   - `setComparisonData([primaryApp, alternativeApp, premiumApp])`
   - `setAppName(primaryApp.app_name)`
   - `isAnalyzing = false` 설정

5. **에러 처리**
   - Toast 알림 표시
   - `isAnalyzing = false` 설정

### 4.3 통계 계산

- 검색 결과가 있을 때만 통계 카드 표시
- `comparisonData` 또는 `analysisResults` 기반으로 계산
- 계산 로직은 분석 결과 화면과 동일

## 5. 인터랙션

### 5.1 검색 인터랙션

1. **입력창 포커스**
   - 포커스 시 테두리 강조
   - 플레이스홀더 텍스트 표시

2. **Enter 키 입력**
   - 키워드가 있고 비활성화 상태가 아니면 검색 실행
   - 검색 중이면 무시

3. **검색 버튼 클릭**
   - 키워드 유효성 검사
   - 검색 실행 또는 에러 알림

4. **예시 키워드 클릭**
   - 해당 키워드가 입력창에 자동 입력
   - 포커스는 입력창에 유지

### 5.2 로딩 상태

- 검색 중: 버튼에 로딩 스피너 표시
- 입력창 및 버튼 비활성화
- 예시 키워드 버튼 비활성화

### 5.3 결과 표시 애니메이션

- 검색 완료 후 결과 섹션이 나타남
- 부드러운 전환 효과 (CSS transition)
